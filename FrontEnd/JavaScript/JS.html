一 作用域

Cbasu

# 一 作用域

可访问变量的集合

# 分类

  1. 全局作用域
  2. 函数作用域
  3. 块级作用域

# 作用域链

当查找变量的时候，会先从当前上下文的变量对象中查找，如果没有找到，就会从父级(词法层面上的父级)执行上下文的变量对象中查找，一直找到全局上下文的变量对象，也就是全局对象。这样由多个执行上下文的变量对象构成的链表就叫做作用域链。

# 函数作用域

函数的作用域在函数定义的时候就决定了
这是因为函数有一个内部属性 [[scope]]，当函数创建的时候，就会保存所有父变量对象到其中，你可以理解 [[scope]] 就是所有父变量对象的层级链，但是注意：[[scope]] 并不代表完整的作用域链！

# js是静态作用域

var value = 1;

function foo() {
    console.log(value);
}

function bar() {
    var value = 2;
    foo();
}

bar();

结果是 1，因为函数的作用域在函数定义的时候就决定了，按照作用域链foo()中没有value则去父级作用域找

# 二 执行上下文

定义：js代码运行环境
分类：全局执行上下文，函数上下文，eval上下文

# 执行栈

执行栈用于存储代码执行期间创建的所有上下文

# 三 this

  1. 默认绑定：函数调用时无任何调用前缀的情景，指向全局对象
  2. 隐式绑定：指向调用函数的那个对象
    如果存在多个对象，this指向距离调用自己最近的对象

    # 隐式丢失（this指向window的情况）

    1. 参数传递
    function fn1(param) {
    param();
};
fn1(obj.fn);
    2)变量赋值
     let fn1 = obj.fn;
 fn1();
  3. 显式绑定：通过call、apply以及bind
    【注意】bind多次执行，函数this还是指向第一次bind的对象。因为bind属于硬绑定，返回的 boundFunction 的 this 指向无法再次通过bind、apply或 call 修改
  4. new
  5. 箭头函数的this:等于它上层上下文中的this

# 四 闭包

  1. 解释：一个可以访问外部作用域的内部函数。
  2. 在本质上，闭包是将函数内部和函数外部连接起来的桥梁。
  3. 闭包中的变量并不保存中栈内存中，而是保存在堆内存中，这也就解释了为什么闭包还能引用到函数内的变量。

# 五 原型

  • 每个构造函数(constructor)都有一个原型对象(prototype),原型对象都包含一个指向构造函数的指针,而实例(instance)都包含一个指向原型对象的内部指针.
  • 如果试图引用对象(实例instance)的某个属性,会首先在对象内部寻找该属性,直至找不到,然后才在该对象的原型(instance.prototype)里去找这个属性.

# 原型链的问题

  • 问题一: 当原型链中包含引用类型值的原型时,该引用类型值会被所有实例共享;
  • 问题二: 在创建子类型(例如创建Son的实例)时,不能向超类型(例如Father)的构造函数中传递参数.

# 六 继承

🔗链接 (opens new window)

# 1. 原型链继承

原理：让新实例的原型等于父类的实例 Child.prototype = new Father()

# 特点

实例可继承的属性有：实例的构造函数的属性，父类构造函数属性，父类原型的属性。（新实例不会继承父类实例的属性！）

# 缺点

  1. 新实例无法向父类构造函数传参。
  2. 继承单一。
  3. 所有新实例都会共享父类实例的属性。（原型上的属性是共享的，一个实例修改了原型属性，另一个实例的原型属性也会被修改！）

# 2. 构造函数继承

原理: 在子类构造函数中使用父类构造函数+call()。

function Child（）{
    Father.call(this.name)
}

# 特点

  1. 只继承了父类构造函数的属性，没有继承父类原型的属性。
  2. 可以继承多个构造函数属性（call多个）。

# 缺点

不能继承父类原型的属性

# 3. 组合继承

# 特点

结合了两种模式的优点，传参和复用

# 缺点

调用了两次父类构造函数（耗内存），子类的构造函数会代替原型上的那个父类构造函数。

# 4. 原型式继承

function content（obj){
    function Child(){} //声明子类的构造函数
    Child.prototype = obj 
    return new Child()  //    返回实例
}
let father = new Father();
let child = content(father)

原理：在object()函数内部, 先创建一个临时性的构造函数, 然后将传入的对象作为这个构造函数的原型,最后返回了这个临时类型的一个新实例。【浅复制】object.create()就是这个原理。

# 特点

类似于复制一个对象，用函数来包装

# 缺点

# 5. 寄生式继承

原理：就是给原型式继承外面套了个壳子。

function content（obj){
    function Child(){} //声明子类的构造函数
    Child.prototype = obj 
    return new Child()  //    返回实例
}
let father = new Father();
function subobject（obj){
    let child = content(father)
    child.name = 'cb'
    return child
}
let child1 = subobject(father)

# 缺点：没用到原型，无法复用。

# 6. 寄生组合式继承

function content（obj){
    function Child(){} 
    Child.prototype = obj 
    return new Child() 
}
let con = content(Father.prototype); //继承原型属性

function Sub（){
    Father.call(this)//继承父类构造函数属性
}
//重点
Sub.prototype = con //继承con的实例
con.constructor = Sub
let child1 = new Sub()
