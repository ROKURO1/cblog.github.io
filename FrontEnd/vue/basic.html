# Basic

# 一.生命周期

钩子函数 描述
beforeCreate vue实例创建后触发
created 初始化data/methods/computed等等完成
beforeMount 模版编译(render)，生成HTML但还没有挂载到DOM上
mounted 挂载结束
beforeUpdate 组件更新前，虚拟DOM打补丁和重新渲染之前
updated
beforeDestory
destoryed

# 二.vue-router

url改变->触发监听事件->改变vue-router中的current变量->（监视current的监视者）监测到变化->获取新的组件->render新组件

# hash

  1. 使用onhashchange监听hash的变化
  2. location.hash直接拿到当前hash

hash只是出现在url中，不会请求http
每次改变#后的值都会在browser历史中加记录

# history

  1. 通过location.pathname拿到
    通过onpopstate 监听history变化
  2. 使用HTML5 history interface新增的pushState和replaceState方法应用于历史记录栈，在back、forward、to基础上对历史记录栈进行修改

# 三.组件间传值

# 父->子

  • 父组件用v-bind绑定
  • 子组件用props接收

# 子->父

  • 子组件使用$emit( )触发父组件事件
  • 父组件接收子组件数据时，接收的方法需绑定在子组件上

# 兄弟间

  • 建立中转站
  • 在两个兄弟组件中都import这个中转站 分别绑定方法用于获取数据
  • 发送方通过emit发送 接收方用on接受

# 四.Computed&watch

# computed 计算属性

作用：自动监听依赖值对变化，从而动态返回内容
特点：由两部分组成，默认只有get
计算属性和methods的区别：计算属性是基于它们的响应式依赖进行缓存的。只在相关响应式依赖发生改变时它们才会重新求值,而不必再次执行函数。

# watch 侦听属性

某个属性发生改变引起对异步或开销较大对操作
handler方法和immdiate属性
immdiate： watch默认绑定，页面首次加载时，是不会执行的。只有值发生改变才会执行。
deep： 如果是监听的是对象类型，会一层层遍历，给这个对象的所有属性都加上这个监听器。但是这样性能开销会比较大，修改任何一个属性，都能触发

# 五.MVC&MVVM

MVC M数据 V视图 C控制器 处理用户交互，从model中取数据传给view中显示或在view中接收用户输入的数据传给model
MVVM 实现了View和Model的自动同步，不需要再去手动操作DOM。

# 六.nextTick

nextTick里面的代码会在DOM更新后执行，因为Vue 异步执行 DOM 更新,并不是数据更新后DOM立刻更新

# 应用场景

一 在created( )中使用 因为created( )中dom没有进行渲染，所以操作dom没有用
二 mounted 中使用nextTick是等待所有视图渲染完毕

# 七.v-if/v-show区别

v-if 元素DOM节点的创建与删除。 涉及到权限、安全
v-show 操作元素的display属性。 需要频繁触发时

# 八.Vuex

集中式状态管理

  • state 读取数据
  • mutations 修改数据,通过this.$store.commit触发
  • getter 类似compute
  • action 将mutations里的方法改成可异步的,通过this.$store.dispatch触发
  • modules 当store对象非常复杂时，可将其分为一个个module（每一个module都包含自己的 state、mutation、action、getter）

# v-bind与v-model区别

# v-bind

  • 是一个***单向数据***绑定，映射关系：Model->View，我们不需要进行额外的DOM操作，只需要进行Model的操作就可以实现视图的联动更新。
  • 用于绑定class等

# v-model

  • 是一个***双向数据***绑定，映射关系：View接受的数据,传给model,model的数据再传给view。把Model绑定到View的同时也将View绑定到Model上，这样就既可以通过更新Model来实现View的自动更新，也可以通过更新View来实现Model数据的更新。
  • 用于输入型元素，如input/select等等
